이 문제가 그리디로 볼 수 있는게 조건이
- 문자열의 뒤에 A를 추가한다.
- 문자열을 뒤집고 뒤에 B를 추가한다.
인데 그럼 둘다 끝이 A아니면 B라는거 그러므로 A아니면 B로 해서 계속 나아가는게 정답이므로 
처음의 선택이 최선인 그리디임
반면 만약에 하나는 끝이 A인게 답일수도 있고 하나는 맨처음이 B인게 답일수도 있는 A와B2 문제의 같은 경우
백트래킹을 통해 A갈래B갈래를 확인해줘야 함

추가로 복습하면서 시간 복잡도에 대해서 생각해보니
t=1000이고 s=1 이라혹 하면 만약 계속 reverse()를 해줘야 하면
1000번
999번
998
.... 계속

1000+999+998+... 등차수열로 나타내보면 n(n+1)/2 최악의 경우 n^2이 된다
여기서 느낀점은 reverse즉 뒤집는 연산은 문자의 길이가 길수록 시간이 길어지기 때문에 뒤집는다라는 말이 나오면
flag 변수를 두거나 n^2일때 통과되는지 생각해봐야 한다
