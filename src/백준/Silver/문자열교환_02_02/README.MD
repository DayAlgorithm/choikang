https://www.acmicpc.net/problem/20437

이 문제는 끝과 처음이 연결된 원형 리스트 구조다. 
윈도우가 배열의 끝을 넘어 맨 앞(0, 1, 2...)까지 이어져야 하는데, 이때 필연적으로 인덱스가 배열의 길이를 초과하게 된다. 
이를 해결하기 위해 (현재인덱스) % 전체길이 공식을 사용하여, 인덱스가 길이를 넘을 때 자동으로 앞쪽으로 돌아오도록 파싱(Parsing) 해야 한다.

1. 문제 상황 (Why?)

이 문제는 문자열의 처음과 끝이 연결된 원형(Ring) 구조입니다.
따라서 슬라이딩 윈도우가 배열의 끝에 도달했을 때, 멈추지 않고 다시 앞쪽(인덱스 0, 1, 2...)으로 넘어가서 검사를 계속해야 합니다.

2. 발생하는 문제 (Problem)

단순히 인덱스를 +1씩 증가시키면, 배열의 최대 길이(len(s))를 초과하게 되어 **IndexError**가 발생합니다.
예: 길이 10인 배열에서 12번 인덱스를 찾으려고 시도함.

3. 해결책 (Solution)

**나머지 연산자 (%)**를 사용하여 인덱스를 회전시킵니다.
인덱스가 길이를 넘어갈 때마다 자동으로 0번부터 다시 시작하게 만드는 "원형 인덱싱(Circular Indexing)" 기법을 적용합니다.
