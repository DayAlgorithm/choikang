### 1. 🧱 왜 '긴 것'을 먼저 넣어야 할까? (테트리스 원리)

달력 문제는 **"빈 공간에 블록 채워 넣기"** 게임입니다.
문제의 조건(우선순위)을 보면 보통 이렇게 되어 있습니다.

1. 시작일이 빠른 순
2. 시작일이 같다면  **기간이 긴 순서**

#### ❌ 만약 짧은 것을 먼저 넣으면? (파편화)

`A(1일~10일)`, `B(1일~2일)` 두 일정이 있다고 칩시다.

* **상황:** 짧은 `B`를 먼저 1번 행(Row 1)에 넣었습니다.
* **결과:** 1번 행의 `1일~2일`이 찼습니다.
* **문제:** 이제 긴 `A(1일~10일)`를 넣으려니, 1번 행 앞부분이 막혀 있습니다.
* **참사:** `A`는 어쩔 수 없이 **2번 행(Row 2)**으로 밀려나게 됩니다.
* **효과:** 달력의 높이가 불필요하게 높아지거나, 직사각형 모양을 계산할 때 로직이 꼬입니다.

#### ✅ 긴 것을 먼저 넣으면? (안정감)

* **상황:** 긴 `A`를 먼저 1번 행에 촥 깝니다.
* **결과:** 1번 행이 든든하게 채워집니다.
* **이후:** 짧은 `B`는 2번 행에 예쁘게 들어갑니다. (혹은 문제 조건에 따라 적절한 위치 배정)
* **핵심:** **"큰 뼈대(긴 일정)"**를 먼저 잡아야 남은 자투리 공간을 효율적으로 관리할 수 있습니다.

### 2. 🐍 코드로 구현하는 법 (Python 람다 정렬)

파이썬의 `sort`는 튜플을 이용해 **다중 조건 정렬**을 기가 막히게 처리합니다.

```python
# 일정 리스트: [시작일, 종료일]
schedules = [[1, 10], [1, 2], [2, 5], [1, 5]]

# 정렬 조건
# 1순위: x[0] (시작일) -> 오름차순 (기본)
# 2순위: x[1] (종료일) -> 내림차순 (긴 게 먼저 와야 하니까!)
#        내림차순을 하려면 '-'를 붙이면 됩니다.

schedules.sort(key=lambda x: (x[0], -x[1]))

print(schedules)
# 결과: [[1, 10], [1, 5], [1, 2], [2, 5]]
# 시작일이 1인 것들 중에서, 종료일이 10 -> 5 -> 2 순서로 예쁘게 정렬됨!

```

### 3. ⚠️ 주의: 문제 유형별 미세한 차이

하지만 모든 일정 문제가 "긴 것 우선"은 아닙니다. **목표**를 잘 봐야 합니다.

#### Type A: 달력 그리기 / 테트리스 (백준 20207)

* **목표:** 예쁘게 쌓기, 위에서부터 채우기.
* **정렬:** 시작일 , **기간  (긴 것 먼저)**
* **이유:** 문제에 명시된 우선순위 규칙 준수.

#### Type B: 회의실 배정 / 최대 개수 (백준 1931)

* **목표:** 최대한 많은 회의 진행하기.
* **정렬:** **종료일  (빨리 끝나는 것 먼저)**, 시작일
* **이유:** 빨리 끝내야 다음 사람을 받을 수 있으니까. (여기선 긴 것 먼저 넣으면 망합니다!)

#### Type C: 강의실 배정 / 최소 개수 (백준 11000)

* **목표:** 최소한의 강의실로 다 소화하기.
* **정렬:** **시작일 **, 종료일
* **이유:** 빨리 시작하는 순서대로 힙(Heap)에 넣고, "가장 빨리 비는 방"과 비교해야 하니까. (여기서는 종료일 순서가 크게 중요하지 않거나 오름차순이 일반적)

### 🧠 Think Ultra 핵심 요약

* 코드는 `sort(key=lambda x: (x[0], -x[1]))` 이거 한 줄이면 끝납니다. 👊
