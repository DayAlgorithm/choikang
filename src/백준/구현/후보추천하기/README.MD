파이썬의 딕셔너리는 자료구조 MAP(해시테이블)이다.
MAP의 내부는 해시함수와 배열로 구현되어 있다
3:[1,3] KEY=3 VALUE[1,3] 이라면
IF 3 IN DIC 로 찾게 되면 키 값인 3을 계산해 인덱스로 즉시 반환해주기 때문에 0(1)의 속도로 찾을 수 있음


**Think Ultra** 레벨로 일반 딕셔너리(`{}`)와 `defaultdict`의 **결정적 차이**를 완벽하게 쪼개드립니다! 🧠🔪

결론부터 말씀드리면: **"일반 `{}`로도 다~ 됩니다! 하지만, 코드가 쓸데없이 길어지고 귀찮아집니다."**

왜 그런지, 님이 가져오신 예시들을 코드로 직접 비교해 보면서 '아하!' 하게 만들어 드릴게요.

---

### 1. ⚔️ 결정적 차이: "없는 키(Key)를 불렀을 때의 태도"

* **일반 `{}`의 태도:** "어? 나한테 없는 키를 달라고? **(에러 쾅! 😡 KeyError)**"
* **`defaultdict`의 태도:** "어? 없는 키네? **내가 알아서 기본값(0이나 빈 리스트)으로 만들어줄게! (천사 👼)**"

이 태도 차이가 코딩 테스트에서 엄청난 길이 차이를 만듭니다.

---

### 2. 🆚 상황별 코드 비교 (왜 defaultdict를 추천하는가?)

#### 🅰️ 개수 세기 (Counting)

사과가 몇 개인지 세어보는 상황입니다.

**❌ 일반 `{}`를 쓸 때 (귀찮음)**

```python
counts = {}
fruit = 'apple'

# 처음 들어온 과일이면 '0'으로 세팅해 주는 "초기화" 코드가 무조건 필요함!
if fruit not in counts:
    counts[fruit] = 0
counts[fruit] += 1 

```

**✅ `defaultdict(int)`를 쓸 때 (마법)**

```python
from collections import defaultdict

# int의 기본값은 '0'이니까, 없는 키가 들어오면 알아서 0으로 세팅해 줌!
counts = defaultdict(int)
fruit = 'apple'

# if문 다 날려버리고 그냥 더하면 끝!
counts[fruit] += 1 

```

#### 🅱️ 그룹 묶기 / 그래프 연결 (Grouping / Graph)

학생들을 반별로 묶거나, 그래프 노드를 연결할 때(리스트 `.append()`)입니다.

**❌ 일반 `{}`를 쓸 때 (귀찮음)**

```python
groups = {}
# 1반에 철수를 넣고 싶은데, 1반 리스트가 아직 없으면 만들어 줘야 함.
if 1 not in groups:
    groups[1] = []  # 빈 리스트 초기화
groups[1].append('철수')

```

**✅ `defaultdict(list)`를 쓸 때 (세상 편함)**

```python
from collections import defaultdict

# list의 기본값은 '[]'(빈 리스트)니까 알아서 만들어 줌!
groups = defaultdict(list)

# if문 없이 그냥 바로 append 때려박기!
groups[1].append('철수')

```

### 3. 🚨 그럼 왜 무조건 `defaultdict`를 안 쓰고 일반 `{}`를 쓸까?

님이 가져오신 표의 마지막 두 줄이 핵심입니다.
**단순 검색 (포켓몬 문제)** 이나 **JSON 데이터**를 다룰 때는 `defaultdict`가 오히려 **독(Poison)**이 됩니다.

#### ☠️ `defaultdict`의 치명적 단점: "침묵의 쓰레기 데이터 생성"

만약 님이 포켓몬 도감을 만들었는데, 누군가 오타를 쳐서 **"피카츄"** 대신 **"피카추"**를 검색했다고 칩시다.

* **일반 `{}`:** 에러(`KeyError`)를 냅니다.  "아, 오타 났구나!" 하고 바로 알 수 있습니다.
* **`defaultdict(int)`:** 에러를 안 냅니다. 대신 도감에 **`"피카추": 0` 이라는 쓰레기 데이터를 몰래 만들어버립니다.** 나중에 도감을 열어보면 듣도 보도 못한 포켓몬들이 잔뜩 들어있게 됩니다.

> **Think Ultra 결론:**
> * 뭔가를 **세거나(`+=1`), 모을 때(`.append()`)**  `defaultdict`가 압도적으로 편하다!
> * 있는지 없는지 **검색만** 하거나, **데이터가 변조되면 안 될 때**  일반 `{}`를 써서 에러를 터뜨려야 안전하다!

### 💡 보너스: `.isdigit()`는 뭔가요?

님이 적어주신 `.isdigit()`는 문자열이 **"숫자로만 이루어져 있는지"** 판별해 주는 파이썬 내장 함수입니다. (포켓몬 문제에서 아주 중요하죠!)

```python
print("123".isdigit())   # True  (정수 맞네!)
print("Pikachu".isdigit()) # False (글자네!)

```

*포켓몬 문제(백준 1620)에서 입력으로 들어온 게 "포켓몬 이름(문자)"인지, "도감 번호(숫자)"인지 구분할 때 이 `.isdigit()`를 쓰면 한 방에 해결됩니다.*

이해가 쏙쏙 되시나요? `defaultdict`의 마법 같은 원리가 이해되셨다면, 포켓몬 문제 풀이의 핵심 구조도 마저 짜드릴까요? 👊
