### 2. ⚡ 왜 검색 속도가  인가? (핵심)

이게 가장 중요합니다. 보통 검색은 "뒤지는 것(Scan)"이라고 생각하지만, 해시 테이블의 검색은 **"계산(Calculation)"**입니다.

* **리스트의 검색 ():**
* "101호에 있나? 아니네. 102호에 있나? 아니네..." (하나씩 확인)


* **해시 테이블의 검색 ():**
* 공식: `해시함수("피카츄") = 101`
* **"야, 그냥 101호로 바로 가."** (확인 과정 없음, 좌표이동)


데이터가 100만 개든 100억 개든, **함수를 한 번 통과해서 나온 숫자(주소)로 텔레포트**하기 때문에 걸리는 시간은 똑같습니다. 그래서 $O(1)$입니다.

### 3. 🧩 해시(Hash) vs 해시 테이블(Hash Table)

이 둘은 바늘과 실 같은 관계지만, 엄연히 다른 개념입니다.

| 개념 | 설명 | 비유 |
| --- | --- | --- |
| **해시 (Hash)** | 데이터를 고정된 길이의 **숫자(또는 문자열)로 변환하는 과정/결과물**. | **"지문"** 또는 **"주민등록번호"** |
| **해시 함수 (Function)** | 그 변환을 수행하는 **수학적 기계**. | **"지문 인식기"** |
| **해시 테이블 (Table)** | 해시 함수를 이용해 데이터를 저장하는 **공간(자료구조)**. | **"지문 인식 사물함"** |

* **해시:** "피카츄"  `25412` (이 숫자가 해시값)
* **해시 테이블:** `Array[25412]`에 데이터를 저장.


### 5. ⚠️ 완벽하진 않다: "해시 충돌(Collision)"

이때 해시 테이블은 두 가지 방법으로 해결합니다.

1. **체이닝(Chaining):** 101호 방 안에 **리스트(연결 리스트)**를 만들어서 꾸역꾸역 같이 넣습니다. (자바가 쓰는 방식)
2. **개방 주소법(Open Addressing):** "어? 찼네?" 하고 옆방(102호)을 봅니다.


### 🚀 Think Ultra 요약

1. **정체:** 해시 테이블은 **"계산기를 달아놓은 배열"**이다.
2. **속도:** 뒤지는 게 아니라 **주소를 계산해서 텔레포트** 하므로 무조건 $O(1)$이다.
3. **관계:** `해시`는 번호를 뽑는 기술이고, `해시 테이블`은 그 번호로 데이터를 관리하는 창고다.
4. **자바:** `HashMap`이 바로 이것이다.
