2. 두번째 방법
값이 들어올때마다 정렬해주고 0일때 빼주기
최악의 경우 절반이 0이라고 하면 N* NlogN의 시간복잡도
   
힙 특성상 작은 값부터 정렬되는데 값이 들어갈때마다 logN의 시간복잡도 그리고
n개가 있으니 NlogN의 시간 복잡도 그리고 힙에 (절댓값,원래값) 이렇게 튜플로 들어가는데
절댓값으로 오름차순 하되, 절댓값이 같다면 뒤에있는 원래값을 비교해 더 작은 값을 맨 앞으로 두게 됨
그러면 자연스럽게 제일 절댓값이 제일 작으면서 원래값도 제일 작은 값이 맨 앞으로 오게되고
pop 연산은 nlog이기에 전체 시간 복잡도는 nlogn+nlogn=nlogn
